{ } JSON - JavaScript Object Notation

JSON (JavaScript Object Notation) ek lightweight data-interchange format hai jo human-readable aur machine-parsable dono hai. Ye web APIs aur configuration files mein sabse zyada use hota hai.

Key Features:

  • Simple Syntax: Key-value pairs mein data store karta hai
  • Language Independent: Almost sab programming languages JSON ko support karte hain
  • Lightweight: Minimal syntax, fast parsing
  • Web Standard: REST APIs ka primary format

Data Types:

  • String - Text values in double quotes
  • Number - Integer ya decimal numbers
  • Boolean - true ya false
  • Array - Ordered list of values
  • Object - Key-value pairs ka collection
  • null - Empty value

Example:

{
  "name": "Rahul Kumar",
  "age": 28,
  "isStudent": false,
  "skills": ["JavaScript", "Python", "React"],
  "address": {
    "city": "Mumbai",
    "state": "Maharashtra",
    "pincode": 400001
  },
  "projects": [
    {
      "title": "E-commerce Website",
      "status": "completed",
      "year": 2023
    },
    {
      "title": "Mobile App",
      "status": "in-progress",
      "year": 2024
    }
  ]
}

Common Use Cases:

  • REST API responses aur requests
  • Configuration files (package.json, tsconfig.json)
  • Data storage aur transfer
  • NoSQL databases (MongoDB)

</> XML - Extensible Markup Language

XML ek markup language hai jo data ko structured aur hierarchical format mein store karta hai. Ye tags use karta hai jo HTML ki tarah dikhte hain lekin khud ke custom tags bana sakte hain.

Key Features:

  • Self-Descriptive: Tags data ko describe karte hain
  • Hierarchical Structure: Parent-child relationships
  • Extensible: Apne custom tags define kar sakte hain
  • Platform Independent: Any system mein use ho sakta hai

Syntax Rules:

  • Must have a root element
  • Tags are case-sensitive
  • Properly nested and closed
  • Attributes in quotes

Example:

<?xml version="1.0" encoding="UTF-8"?>
<employee>
  <name>Rahul Kumar</name>
  <age>28</age>
  <department id="IT-001">
    <name>Information Technology</name>
    <location>Mumbai</location>
  </department>
  <skills>
    <skill level="expert">JavaScript</skill>
    <skill level="intermediate">Python</skill>
    <skill level="expert">React</skill>
  </skills>
  <projects>
    <project status="completed">
      <title>E-commerce Website</title>
      <year>2023</year>
    </project>
  </projects>
</employee>

Common Use Cases:

  • SOAP web services
  • Configuration files (web.xml, pom.xml)
  • Document storage (Office files)
  • RSS feeds aur sitemaps

üìÑ YAML - YAML Ain't Markup Language

YAML ek human-friendly data serialization format hai jo configuration files ke liye bahut popular hai. Ye indentation use karta hai structure define karne ke liye, brackets ya braces ki zarurat nahi hoti.

Key Features:

  • Human Readable: Sabse easy to read format
  • Indentation-Based: Whitespace se structure define hota hai
  • No Brackets: Clean aur minimal syntax
  • Comments Support: # se comments add kar sakte hain

Syntax Features:

  • Colon (:) for key-value pairs
  • Hyphen (-) for lists
  • Indentation (2 or 4 spaces) for nesting
  • Quotes optional for strings

Example:

# Employee Configuration
name: Rahul Kumar
age: 28
isStudent: false

# Skills List
skills:
  - JavaScript
  - Python
  - React

# Nested Address Object
address:
  city: Mumbai
  state: Maharashtra
  pincode: 400001

# Array of Objects
projects:
  - title: E-commerce Website
    status: completed
    year: 2023
    technologies:
      - React
      - Node.js
      - MongoDB

  - title: Mobile App
    status: in-progress
    year: 2024
    technologies:
      - React Native
      - Firebase

Common Use Cases:

  • Docker Compose files
  • Kubernetes configurations
  • CI/CD pipelines (GitHub Actions, GitLab CI)
  • Application configuration files

‚öôÔ∏è TOML - Tom's Obvious Minimal Language

TOML ek configuration file format hai jo INI files se inspired hai. Ye human-readable hai aur unambiguous parsing ke liye design kiya gaya hai. Rust aur Python projects mein bahut popular hai.

Key Features:

  • Obvious Semantics: Clear aur unambiguous syntax
  • Minimal: Simple aur easy to parse
  • Type Safety: Strong typing support
  • Comments: # se comments support karta hai

Data Types:

  • String, Integer, Float, Boolean
  • Datetime
  • Arrays
  • Tables (objects)

Example:

# Employee Configuration File

name = "Rahul Kumar"
age = 28
isStudent = false

# Skills Array
skills = ["JavaScript", "Python", "React"]

# Address Table
[address]
city = "Mumbai"
state = "Maharashtra"
pincode = 400001

# Array of Tables - Projects
[[projects]]
title = "E-commerce Website"
status = "completed"
year = 2023
technologies = ["React", "Node.js", "MongoDB"]

[[projects]]
title = "Mobile App"
status = "in-progress"
year = 2024
technologies = ["React Native", "Firebase"]

# Nested Table Example
[database]
server = "192.168.1.1"
port = 5432
connection_max = 5000
enabled = true

[database.credentials]
username = "admin"
password = "secure_password"

Common Use Cases:

  • Cargo.toml (Rust package manager)
  • pyproject.toml (Python projects)
  • Application configuration files
  • Build system configurations

üé® Format Function

Format function data ko human-readable format mein convert karta hai. Ye proper indentation aur line breaks add karta hai taaki code easy to read ho jaye.

Purpose:

Minified ya unformatted data ko properly indented aur readable banata hai

How it Works:

  "name": "Rahul",
  "age": 28,
  "skills": [
    "JS",

üì¶ Minify Function

Minify function data ko compress karta hai by removing unnecessary whitespace, line breaks, aur indentation. Ye file size ko reduce karta hai jo network transfer ke liye beneficial hai.

Purpose:

File size reduce karke faster loading aur less bandwidth usage

How it Works:

  • Sab whitespace remove karta hai
  • Line breaks aur indentation hatata hai
  • Data integrity maintain rakhte hue size minimize karta hai

Before Minify (Formatted):

{
  "name": "Rahul Kumar",
  "age": 28,
  "skills": [
    "JavaScript",
    "Python",
    "React"
  ]
}

After Minify:

{"name":"Rahul Kumar","age":28,"skills":["JavaScript","Python","React"]}

Benefits:

  • Reduced file size (often 40-60% smaller)
  • Faster network transfer
  • Less bandwidth usage
  • Improved load times
Compress Optimize Reduce Size

üîß Repair Function

Repair function broken ya invalid data ko fix karta hai. Common syntax errors ko automatically detect aur correct karta hai.

Purpose:

Invalid syntax wale data ko valid format mein convert karna

Common Issues it Fixes:

  • Missing commas between elements
  • Trailing commas
  • Missing quotes around strings
  • Unclosed brackets or braces
  • Single quotes instead of double quotes (JSON)

Example 1 - Missing Commas:

Before Repair:

{
  "name": "Rahul"
  "age": 28
  "city": "Mumbai"
}

After Repair:

{
  "name": "Rahul",
  "age": 28,
  "city": "Mumbai"
}

Example 2 - Single Quotes:

Before Repair:

{'name': 'Rahul', 'age': 28}

After Repair:

{"name": "Rahul", "age": 28}

üîÑ Format Conversion Functions

1. JSON to XML Conversion

JSON data ko XML format mein convert karta hai. Objects XML elements ban jate hain aur arrays repeated elements ban jate hain.

Example:

JSON Input: 
{
  "employee": {
    "name": "Rahul",
    "age": 28,
    "skills": ["JavaScript", "Python"]
  }
}
XML Output: 
<employee>
  <name>Rahul</name>
  <age>28</age>
  <skills>JavaScript</skills>
  <skills>Python</skills>
</employee>

2. XML to JSON Conversion

XML data ko JSON format mein convert karta hai. Elements objects ban jate hain aur repeated elements arrays ban jate hain.

Example:

XML Input: 
<person>
  <name>Rahul</name>
  <age>28</age>
</person>
JSON Output: 
{
  "person": {
    "name": "Rahul",
    "age": "28"
  }
}

3. JSON to YAML Conversion

JSON ko YAML's human-friendly syntax mein convert karta hai.

Example:

JSON Input: 
{"name": "Rahul", "age": 28, "skills": ["JS", "Python"]}
YAML Output: 
name: Rahul
age: 28
skills:
  - JS
  - Python

4. JSON to TOML Conversion

JSON data ko TOML configuration format mein convert karta hai.

Example:

JSON Input: 
{
  "database": {
    "server": "localhost",
    "port": 5432
  }
}
TOML Output: 
[database]
server = "localhost"
port = 5432

6. XML to TOML Conversion

üí° Pro Tips & Best Practices

JSON Best Practices:

  • Always use double quotes for strings
  • No trailing commas allowed
  • Use consistent indentation (2 or 4 spaces)
  • Validate JSON using online validators

XML Best Practices:

  • Use meaningful tag names
  • Close all tags properly
  • Use attributes for metadata only
  • Include XML declaration at top

YAML Best Practices:

  • Use 2 spaces for indentation (not tabs)
  • Quote strings with special characters
  • Use comments to explain complex sections
  • Keep nesting levels reasonable

TOML Best Practices:

  • Group related configuration together
  • Use tables for nested data
  • Prefer arrays of tables for repeated data
  • Add comments for important settings
